Examples > Coding > javafx
Items in one sentence about javafx
Tags:Need supplementary;;;0xfff2dfff;;;Like;;;0x0d3928ff
<ol>
    <li>Always assume user knows nothing about programming.</li>
    <li>Do not expect user will open user guide. Someone, including me, even will not look at nor think about prompts on the screen.</li>
    <li>Necessary prompts can help user understand intention of the program.</li>
    <li>Ambiguous prompt is worse than no prompt.</li>
    <li>Always response for user, like pop information, make sound, or update interface.</li>
    <li>Provide most options and set default choice.</li>
    <li>Remember user’s inputs or seletions and fill in or set values automatically in next time.</li>
    <li>Less inputs or selections meanwhile keep interfaces clean.</li>
    <li>Use radio or check buttons instead of drop-down list if there is enough place in interface.</li>
    <li>No more than 3 levels to select.</li>
    <li>When too many controls in interface, consider popular layout:  Left-right areas like curtain, vertical accordion menus, tabs to switch targets</li>
    <li>User’ screen resolution may be from 1k to 4k, so the design size of most MyBox interfaces is smaller than 1100 * 720.</li>
    <li>Make use of auto-wrap container like FlowPane, to work for both small screen and large screen.</li>
    <li>Controls can be updated only in JavaFx thread. Use Platform.runLater to change control when handle event in its listener.</li>
    <li>Progress should be shown when operation costs much time.</li>
    <li>Time-consume operation should not be run in JavaFx thread to avoid blocking interface.</li>
    <li>Backgournd task should always check whether task is canceled, especially in loop or before/after long-run statements.</li>
    <li>When listen mouse events, event.getX() and event.getY() return the coordinate of event.getSource() which is the control owns the events. Example, when listen on ImageView, the coordinate is related to the image.</li>
    <li>Iterative calls may happen when listen both onMouseEntered and  onMouseExited and change focus in handling of the events(like pop menu) or change control’s size.</li>
    <li>When double click mouse, single-click event is tiggerred at first and then double-click event is triggerred. So need avoid duplicated or conflicted handlings against same event.</li>
    <li>For table's multiple selections, use getSelectedIndices() to listen changes, Some changes may be missed if use selectedItemProperty().</li>
</ol>

Examples > Coding > java
Items in one sentence about java
Tags:Need supplementary;;;0xfff2dfff;;;Like;;;0x0d3928ff
<ol>
    <li>Always assume most extreme inputs and most weird operations.</li>
    <li>Assume user’ hardware configuration(CPU/memory/disk) is lower than average level.</li>
    <li>Use BufferedInputStream and BufferedOutputStream for file I/O.</li>
    <li>Use try-with-resource for file I/O.</li>
    <li>Always consider out of memory when read/write file or data.</li>
    <li>Notice charset when read/write text file.</li>
    <li>Notice charset when convert between bytes and string.</li>
    <li>Condition to determine whether reach end of file should be "len &gt; 0". If use "len != -1", endless loop may happen if zero byte is read.</li>
    <li>Call "System.gc()" before delete/move/rename file, to avoid failure due to cache.</li>
    <li>When handle information in tree, like files system, endless loop should be avoided. 
If the operation will change source node, the target node should not be the source node nor its descendants.</li>
    <li>If delete list by indices, delete items from the end index to the start index.</li>
    <li>When class has many parameters, do not write many constructors to initialized different sets of parameters. Better coding is static creator and chain assignment by return “this” in all “set” methods.</li>
    <li>Anonymous class is very useful for scenario like lots of duplicated codes with only difference that  several methods have difference implements.</li>
    <li>Generic is very useful for scenario like lots of duplicated codes with only differernce of data type.</li>
    <li>Define as less instance variables/methods as possible in class if it generates lots of instance. </li>
    <li>As less float-point calculations as possible in loop. If can, change float-point calculations as integer calculations.</li>
    <li>When listen data changing, need judge whether change trully happens, and avoid too frequently manufactures by setting minimum threshold of changing.</li>
    <li>Reboot application when change JVM parameters or need clean env in simple way.</li>
    <li>Notice whether the index is 0-based or 1-based, especially for thrid-party codes. 0-based is default. If 1-based, should write comment for it.</li>
    <li>Generally, index in user interface starts from 1, while index in codes data starts from 0.</li>
    <li>Generally, range(start-end) in user interface includes end, while range(start-end) in codes data excludes end.</li>
    <li>Double.MIN_VALUE is the minimum positive double(almost zero). -Double.MAX_VALUE is the minimum double.</li>
    <li>When handle money, store integer numbers for each unit.</li>
    <li>BigDecimal is much slower than double. Double is enough for most situations.</li>
    <li>To determine value's broken, write if (abs(x - 7.3) &lt; TOLERANCE) instead of if (x == 7.3).</li>
    <li>Value of final static constant should be determined at compling time.</li>
    <li>When JDBC read numbers, if SQL value is null, ResultSet.getDouble/getFloat/getInt/getShort returns 0. Use ResultSet.getObject instead if need distinct zero and null.
       <a target=_blank href="https://docs.oracle.com/en/java/javase/18/docs/api/java.sql/java/sql/ResultSet.html#getDouble(java.lang.String)">ResultSet.html#getDouble</a>
    </li>
    <li>From Java18, Charset.defaultCharset() is always UTF-8. This will affect commands and results of ProcessBuilder. 
        <a target=_blank href="https://docs.oracle.com/en/java/javase/18/docs/api/java.base/java/lang/Process.html#inputReader()">Process</a> 
        <a target=_blank href="https://bugs.openjdk.org/browse/JDK-8266075">JDK-8266075</a>
     </li>
